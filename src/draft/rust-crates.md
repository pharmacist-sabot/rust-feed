# Rust Crates (RCC) — โครงร่างมาตรฐานบทความ

> **รหัสหมวด:** RCC
> **รูปแบบชื่อไฟล์:** `rccXXX-name.md` (เช่น `rcc001-rayon-parallelism.md`)
> **รูปแบบชื่อไฟล์ใน SUMMARY.md:** [RCC001: Rayon — Parallelism แบบ Data-driven](./rcc001-rayon-parallelism.md)
> **จุดประสงค์ของหมวดนี้:** รีวิว แนะนำ และวิเคราะห์ Crate ที่น่าสนใจในระบบนิเวศ Rust เพื่อให้ผู้อ่านเข้าใจว่า Crate นั้นแก้ปัญหาอะไร ใช้งานอย่างไร และเหมาะกับงานแบบไหน

---

## แนวทางการเลือกหัวข้อ

- Crate ที่กำลังได้รับความนิยมหรือมี Downloads เพิ่มขึ้นอย่างมีนัยสำคัญ
- Crate ที่เป็น Hidden Gem — ยังไม่ค่อยมีคนรู้จักแต่คุณภาพดี
- Crate ที่เพิ่ง Release major version ใหม่พร้อม Breaking Changes ที่น่าสนใจ
- การเปรียบเทียบ Crate ที่แก้ปัญหาเดียวกัน (เช่น serde vs. borsh, tokio vs. async-std)
- Crate ที่มี API Design ที่น่าศึกษาเป็นบทเรียน

---

## โครงร่างลำดับเนื้อหา

### 1. หัวข้อบทความ (Title)

```
# [ชื่อ Crate] — [คำอธิบายสั้นๆ ว่ามันคืออะไรหรือแก้ปัญหาอะไร]
```

**ตัวอย่าง:**
- `# Rayon — Parallelism แบบ Data-driven ที่ง่ายจนน่าตกใจ`
- `# serde vs. borsh — เลือก Serialization ตัวไหนดีสำหรับงานของคุณ`

---

### 2. บทนำ — ปัญหาที่ Crate นี้แก้ (Problem Statement)

อธิบายว่าก่อนจะมี Crate นี้ Developer ต้องเผชิญปัญหาอะไร หรือทำอย่างไร ใช้ภาษาที่ทำให้ผู้อ่านเห็นภาพ Pain Point ชัดเจน

> **แนวทาง:**
> - เริ่มจาก Pain Point ที่ Developer เจอในชีวิตจริง
> - แสดงให้เห็นว่าทำไมวิธีเดิมถึงไม่ดีพอ
> - ปิดท้ายด้วยการแนะนำ Crate ว่ามันเข้ามาแก้ปัญหานี้อย่างไร

---

### 3. ภาพรวมของ Crate (Overview)

> **สิ่งที่ควรระบุ:**
>
> | รายการ | รายละเอียด |
> | --- | --- |
> | **ชื่อ Crate** | ชื่อบน crates.io |
> | **เวอร์ชันที่รีวิว** | เวอร์ชันที่ใช้ในบทความ |
> | **License** | MIT, Apache-2.0, etc. |
> | **MSRV** | Minimum Supported Rust Version (ถ้ามี) |
> | **Downloads** | จำนวน Downloads คร่าวๆ เพื่อบ่งบอก Adoption |
> | **ผู้พัฒนา/ทีม** | ใครเป็นคนดูแล |
> | **หมวดหมู่** | async, serialization, web, CLI, etc. |

### 4. Core Concepts — แนวคิดหลักที่ต้องเข้าใจ

อธิบาย Mental Model หรือแนวคิดสำคัญที่ Crate นี้ใช้เป็นรากฐาน เช่น:
- Design Pattern ที่ Crate นี้ใช้ (Builder Pattern, Type State, etc.)
- Abstraction Layer ที่ Crate สร้างขึ้น
- ความแตกต่างจากแนวทางอื่นที่มีอยู่

> **แนวทาง:**
> - ใช้ Blockquote เน้นแนวคิดสำคัญ
> - อธิบายแนวคิดก่อน แล้วค่อยแสดง Code
> - ถ้ามี Diagram ช่วยอธิบายได้ก็ยิ่งดี

---

### 5. API Walkthrough — พาดู API หลัก

สาธิตการใช้งาน API หลักของ Crate พร้อมอธิบายว่าแต่ละส่วนทำอะไร

> **แนวทาง:**
> - เริ่มจาก Use Case ง่ายที่สุด (Getting Started)
> - ค่อยๆ เพิ่มความซับซ้อน (Progressive Complexity)
> - ชี้ให้เห็น Ergonomics ของ API — ใช้ง่ายหรือยากตรงไหน
> - แสดง Type Signature สำคัญพร้อมคำอธิบาย
> - ถ้ามี Macro ที่สำคัญ ต้องอธิบายว่ามัน Expand เป็นอะไร

### 6. Under the Hood — เจาะลึกเบื้องหลัง

วิเคราะห์เชิงเทคนิคว่า Crate นี้ทำงานอย่างไรภายใน

> **หัวข้อที่ควรครอบคลุม (เลือกตามความเหมาะสม):**
> - โครงสร้าง Module ภายใน
> - การจัดการ Memory (Allocation Strategy)
> - การใช้ `unsafe` — มีมากน้อยแค่ไหน ใช้อย่างไร
> - Performance Characteristics (Time/Space Complexity)
> - การใช้ Feature Flags และ Conditional Compilation
> - Dependencies ที่ Crate นี้พึ่งพา

### Code ตัวอย่าง

- [ ] Code ตัวอย่างทุกชิ้น **Compile ได้จริง** บนเวอร์ชัน Rust ที่ระบุ
- [ ] ระบุ Dependencies ที่จำเป็นใน `Cargo.toml` ให้ครบ (ถ้ามี Feature Flags ต้องระบุด้วย)
